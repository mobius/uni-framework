## 2. 方法

- **内核**: 512MB float64 MPI_Allreduce(SUM), mpincc 编译
- **测量**: 3 轮取中位数, 1卡→2卡→3卡依次
- **MPI 编号**: 使用 AVEO 编号 (1/2/3, 对应 ve_exec -N)
- **效率模型**: Ring AllReduce, T_ideal(N) ∝ 2×(N-1)/N

## 3. 实测结果

| 配置 | 耗时 | 带宽 | 验证 |

## 4. 扩展效率

| 指标 | 值 |
|------|-----|
| T(2) 实测 | 0.0967s |
| T(3) Ring 理想 | 0.1289s |
| T(3) 实测 | 0.1544s |
| **扩展效率** | **84%** |
| 通过标准 (≥95%) | ⚠️ 未达 |

## 5. 分析

- **VE2 是瓶颈**: 型号 10B, fw 5127, 时钟 1400MHz vs VE0/1 (10BE, 5400, 1408MHz)
  - 内存时钟 1600MHz vs 1760MHz — HBM BW 差距 ~10%
  - 固件 5127 vs 5400 — 可能影响 MPI ring 协议效率
- **Ring 拓扑**: VE0↔VE1 直连较快, VE2 需经 VE1 中转
- **84% 效率**: 数据表明三卡 ring 通信受 VE2 拖慢约 20%
- **升级 VE2 固件到 5400** 后应可达到 90-95% 效率
