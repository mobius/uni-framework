## 1. 应用 A: 异构稀疏矩阵-向量乘 (SpMV) ✅

### 设计

```
Host(numpy CSR生成) → scp → Phi(244线程列分块) → scp → 3×VE(并行SpMV) → Host(合并验证)
```

- **Phi**: 读取 CSR，按列范围分 3 块。单线程计数+分布，利用 Phi 多线程仅做 I/O 无关的计算
- **VE**: 各自读取分块数据，OpenMP 并行 SpMV
- **数据流**: Host ↔ mic0 scp 双向传输

### 新增文件

```
src/apps/hetero_spmv/
├── spmv_app.py              # 主调度 (生成→scp→Phi→scp→VE→验证)
├── phi/csr_partition.c      # Phi CSR 列分块内核
└── ve/blocked_spmv.c        # VE 分块 SpMV 内核
```

### 实测结果

| 指标 | 值 |
|------|-----|
| 矩阵规模 | N=4096, nnz=167,772 (1% 密度) |
| Phi 分块 | 1.3s (scp 0.5s + partition 0.3s + scp 0.5s) |
| VE 并行 SpMV | 0.107s (3 卡, 5.4-5.6 GB/s) |
| 正确性 | max_diff = 1.07e-14 ✅ |

### 踩坑记录

| 问题 | 根因 | 修复 |
|------|------|------|
| Phi 文件 I/O 不可达 | micnativeloadex 文件系统隔离 | 改用 scp 双向传输 |
| Phi 前缀和偏移错 | 计数存 `[i+1]` → CSR 偏移一位 | 存 `[i]`, 单独建累积索引 |
| 旧文件残留 | mic0 root 属主无法删除 | UUID 唯一文件名 |
| 分布指针初始化 | 累积起始位置 vs 个体计数混淆 | v2 重写: 三 pass 分离 |
