## 实验数据

### 单次对比 (N=512)

| 实现 | GFLOPS | 峰值占比 | 相对 naive |
|------|--------|---------|-----------|
| Naive ikj | 64 | 3.0% | 1× |
| **NLC BLAS** | **1,122** | **51.9%** | **17.5×** |

### 规模化 (N=2048)

| 实现 | GFLOPS | 峰值占比 |
|------|--------|---------|
| Naive ikj | 157 | 7.3% |
| **NLC BLAS** | **1,711** | **79.2%** |

> N=2048 时 NLC 接近理论峰值 (2,160 GFLOPS) 的 79%，与文档中 81% 一致。

### 全框架基准 (bench_all)

```
设备      操作          预估算力    实测算力    效率
────────────────────────────────────────────────
ve      dgemm (NLC)   1750       1122        64%  ← N=512, 小矩阵
ve      dgemm (NLC)   1750       1119        64%
ve      dgemm (NLC)   1750       1109        63%
ve      dgemm (NLC)   1750       1124        64%
```

N=512 时 NLC 效率 64% (较 N=2048 的 98% 低)，因为小矩阵的 cache blocking 收益有限 + ve_exec 启动开销占比较高。

## 用法

```bash
# NLC 内核编译
ncc -O3 -fopenmp -o dgemm_nlc_ve dgemm_nlc.c \
    -I/opt/nec/ve/nlc/3.1.0/include \
    -L/opt/nec/ve/nlc/3.1.0/lib -lcblas -lblas_openmp

# 运行
VE_LD_LIBRARY_PATH=/opt/nec/ve/nlc/3.1.0/lib \
    ve_exec -N 1 ./dgemm_nlc_ve input.bin output.bin
```

## 结论

NLC BLAS 将 DGEMM 性能从 64 GFLOPS 提升到 1,122-1,711 GFLOPS，是框架中**单项最大收益的优化**。N=2048 时逼近理论峰值 79%。下一个 P0 优化项是 Phi 启动摊还。
