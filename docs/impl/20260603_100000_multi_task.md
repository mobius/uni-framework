## 2. 核心组件: TaskGraph 调度器

`src/scheduler/task_graph.py` 实现了一个与设备无关的 DAG 调度器：

- `TaskNode`: 封装任务名、目标设备、执行函数、依赖列表、运行时状态
- `TaskGraph`: 负责任务添加、拓扑排序、条件并行执行
- 自动检测就绪任务（依赖满足）并并行启动
- 死锁检测
- 耗时统计和结果汇总

---

## 3. 任务流定义

```
Task0 (Host) ─┬── Task1 (VE1) ──┐
              ├── Task2 (VE2) ──┼── Task4 (VE1) ── Task5 (Host)
              └── Task3 (VE3) ──┘
```

| 任务 | 设备 | 说明 |
|------|------|------|
| gen | Host (Python) | 生成 3 对随机矩阵 A_i, B_i (512×512) |
| matmul_1 | VE1 | DGEMM: C1 = A1 × B1 |
| matmul_2 | VE2 | DGEMM: C2 = A2 × B2 |
| matmul_3 | VE3 | DGEMM: C3 = A3 × B3 |
| aggregate | VE1 | 聚合: C = C1 + C2 + C3 |
| stats | Host (Python) | 统计: min/max/mean/stddev |

## 6. 已知局限

- Phi stats 内核通过 scp+ssh 执行时遇到文件 I/O 兼容性问题，暂时改为 Host 端 Python 实现
- matmul_block 使用 naive i-k-j DGEMM，性能 (~63 GFLOPS) 远低于 NLC BLAS (~1750 GFLOPS)
- 数据传递使用文件系统，非零拷贝

## 7. 与 Basic 的对比

| 维度 | Basic | Multi-Task |
|------|-------|-----------|
| 任务数 | 4 独立 | 6 有依赖 |
| 调度模式 | asyncio.gather | DAG 拓扑排序 |
| 数据流 | 无 | 文件传递 (binary) |
| 负载类型 | 同构 FMA | 生成/DGEMM/聚合/统计 |
| 设备参与 | Phi + 3VE | 3VE + Host |
