# 串行流水线规划

> 日期: 2026-06-03
> 依赖: Basic + Multi-Task 通过

---

## 目标

实现一个严格的**串行依赖链**：每步的输出作为下一步的输入，数据流经全部 5 个计算单元。

```
Host(gen) → VE1(dgemm) → VE2(scale) → VE3(transpose) → Phi(stats) → Host(report)
```

与 Multi-Task 的区别：

| 维度 | Multi-Task | Pipeline |
|------|-----------|---------|
| 拓扑 | 分叉-汇合 | **严格串行链** |
| 并行度 | 3 VE 并行 | 0（同时只有 1 个设备活跃） |
| 数据流 | 独立输入 | **上游→下游接力** |
| 演示目的 | DAG 并行调度 | **全设备串联数据管道** |

## 流水线定义

| 步骤 | 设备 | 内核 | 输入 | 输出 | 操作 |
|------|------|------|------|------|------|
| gen | Host | Python | — | matrix_A.bin, matrix_B.bin | 生成 512×512 随机矩阵 A,B |
| dgemm | VE1 | matmul_block_ve | matrix_A.bin, matrix_B.bin | C1.bin | C1 = A × B |
| scale | VE2 | scale_ve | C1.bin | C2.bin | C2 = C1 × 2.0 |
| transpose | VE3 | transpose_ve | C2.bin | C3.bin | C3 = C2^T |
| stats | Phi | peak_fp64.mic | C3.bin (scp) | stats.txt | min/max/mean/stddev |
| report | Host | Python | stats.txt | — | 打印最终报告 |
