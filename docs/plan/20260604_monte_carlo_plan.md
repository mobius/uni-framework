# 应用 C: 异构 Monte Carlo 期权定价 — 详细计划

## 1. 背景

### 金融场景

**亚式障碍看涨期权 (Asian Barrier Call)**:

- 标的资产 S_t 服从几何布朗运动: dS = μS·dt + σS·dW
- 到期日 T=1 年, 252 个时间步
- 执行价 K=100, 初始价 S0=100 (平值)
- 障碍价 B=80: 任意时刻 S_t < B → down-and-out, 期权作废
- 亚式: payoff = max(avg(S_t) - K, 0), avg 是路径均价

这是一类没有解析解的复杂期权，必须用 Monte Carlo 模拟定价。

### 为何适合 Phi+VE

| 子任务 | 设备 | 计算特征 |
|--------|------|---------|
| 随机路径生成 + barrier 检测 | **Phi** | 500K 路径独立, 244 线程并行; barrier 是分支判断 |
| payoff 折现累加 | **3×VE** | 大量浮点归约, HBM 带宽 + 向量化 |

## 2. 问题

- 纯 CPU 做 500K×252 步模拟 → 耗时长
- 纯 Phi 做完整流程 → 统计归约不是 Phi 强项
- 纯 VE 做路径生成 → 无分支优化 + 随机数生成效率低

**异构分拆**: Phi 承担不规则部分 (分支+随机), VE 承担规则部分 (归约)

## 3. 解决方法

### 架构

```
Host: S0=100, K=100, B=80, σ=0.2, r=0.05, T=1, N_paths=500K
  ↓ scp 参数文件
Phi: 路径生成 + barrier 检测 → 输出每条有效路径的均价
  ↓ scp
3×VE: 各 1/3 路径, payoff = max(avgS-K, 0) × exp(-rT)
  ↓
Host: 价格 = mean(payoffs) ± 1.96·σ/√N (置信区间)
```

### 文件布局

```
src/apps/monte_carlo/
├── mc_app.py              # 主调度
├── phi/
│   └── path_gen.c          # Phi: 路径生成 + barrier 检测
└── ve/
    └── payoff.c            # VE: payoff 计算 + 折现
```

## 4. 结果验证

| 验证方式 | 方法 | 标准 |
|----------|------|------|
| 退化用例 | B=0 (无障碍), payoff=max(S_T-K,0) = 欧式期权 | vs Black-Scholes 解析解, 误差 <0.5% |
| 参考实现 | Host numpy 相同随机种子完整 MC | 逐路径对比 |
| 收敛性 | N=10K/50K/500K | 价格收敛到稳定值 |
