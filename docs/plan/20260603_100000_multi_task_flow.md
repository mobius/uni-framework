# 多卡异构任务流规划

> 日期: 2026-06-03
> 依赖: V1 Basic 验证通过

---

## 目标

在 Basic 验证（独立并行）基础上，实现一个**有依赖关系的异构任务流**：

1. **DAG 任务图** — 任务按拓扑顺序执行，无依赖的任务并行
2. **异构分发** — 不同任务分配到 Phi 或 VE，依据任务特征
3. **数据传递** — 上游任务产出 → 下游任务消费（通过文件）
4. **混合负载** — 同一时刻 Phi 和 VE 执行不同类型计算

## 模拟场景

```
         ┌──────────────────┐
         │  Task 0: 数据生成  │  Host (Python)
         │  随机矩阵 4096²    │
         └────────┬─────────┘
                  │
        ┌─────────┼─────────┐
        │         │         │
   ┌────┴───┐ ┌───┴───┐ ┌───┴───┐
   │Task 1  │ │Task 2 │ │Task 3 │  VE1/2/3 并行
   │DGEMM   │ │DGEMM  │ │DGEMM  │  矩阵分块乘法
   │Block A │ │Block B│ │Block C│
   └───┬────┘ └───┬───┘ └───┬───┘
       │          │         │
       └──────────┼─────────┘
                  │
         ┌────────┴─────────┐
         │  Task 4: 结果聚合 │  VE1 (AllReduce)
         │  校验和 + 汇总    │
         └────────┬─────────┘
                  │
         ┌────────┴─────────┐
         │  Task 5: 特征统计 │  Phi
         │  均值/方差/最大值  │
         └──────────────────┘
```

> 这是模式 B (流水线型) 的简化版：VE 主力计算 → Phi 后处理。

## 任务定义

| 任务 | 设备 | 内核 | 输入 | 输出 |
|------|------|------|------|------|
| Task 0: gen_data | Host | Python numpy | — | matrix_A.dat, matrix_B.dat |
| Task 1: dgemm_A | VE1 | ve_matmul (ncc) | matrix_A[:2048], B[:2048] | result_1.dat |
| Task 2: dgemm_B | VE2 | ve_matmul (ncc) | matrix_A[2048:], B[2048:] | result_2.dat |
| Task 3: dgemm_C | VE3 | ve_matmul (ncc) | matrix_A[:2048], B[2048:] | result_3.dat |
| Task 4: aggregate | VE1 | ve_aggregate | result_1/2/3.dat | final_result.dat |
| Task 5: stats | Phi | phi_stats | final_result.dat | stats_report.txt |

## DAG 依赖图

```
Task0 (Host) ─────┬──→ Task1 (VE1) ──┐
                  ├──→ Task2 (VE2) ──┼──→ Task4 (VE1) ──→ Task5 (Phi)
                  └──→ Task3 (VE3) ──┘
```

## 通过标准

| 指标 | 期望 |
|------|------|
| DAG 正确执行 | 拓扑顺序，依赖满足 |
| 并行度 | Task1/2/3 真正并行 (执行时间 ≈ max(单卡)) |
| 数据一致性 | 校验和匹配 |
| 总执行时间 | < 30 秒 |
| 无错误 | 全部 PASS |

## 与 Basic 的差异

| 维度 | Basic | Multi-Task |
|------|-------|-----------|
| 任务关系 | 4 个独立任务 | 6 个有依赖关系的任务 |
| 调度模式 | asyncio.gather | DAG 拓扑排序 + 条件并行 |
| 数据流 | 无 (各自闭环) | 上游→下游通过文件传递 |
| 设备分工 | 全部跑相同内核 | 不同设备跑不同内核 |
| 负载特征 | 同构 FMA | 混合: 生成/DGEMM/聚合/统计 |
